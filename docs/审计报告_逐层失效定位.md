>
> **验证级别标注**：🟢 多轮验证 (≥2 independent runs + paired bootstrap) | 🟡 单轮验证 (1 run, 待复现) | 🔴 待验证

## ⚠️ 重要更正 (2026-07-14 晚)

**本报告全部实验均在 PPO ratio 非法状态下完成。**

代码审计发现了一个 P0 级 bug：PPO 更新阶段 `evaluate_actions()` 调用 `actor(obs)` 时**没有传入 rollout 期间使用的 GRU hidden state**。这意味着：

- rollout 时： π(a_t | o_t, h_{t-1})  ← GRU 有累积时序状态
- 更新时：   π(a_t | o_t, h=0)        ← GRU 从零初始化

PPO ratio r_t = π_new / π_old 在第一次 optimizer step 之前就已经 ≠ 1。

**2026-07-14 晚已修复**（commit `7993770` / `b22c7df`），修复内容：
1. Buffer 存储每帧 GRU hidden state `h_prev` (K, K-1, D)
2. PPO 更新时从 buffer 读取并传入 `evaluate_actions()`
3. 新增 `verify_old_log_prob_consistency()` 断言（训练开始时自动执行）
4. Attention 冻结补全（`attn_norm.*` 之前被遗漏）
5. Q=8 硬编码移除

**修复后验证结果（第一轮，`test_ppo_ratio_fix.py` v1）**：
| | steady_P_D | weak3 | worst |
|---|---|---|---|
| DAgger baseline (h=0 eval) | 0.6479 | 0.1795 | 0.0252 |
| Full PPO 1 upd (修复后) | 0.6498 | 0.1836 | 0.0188 |
| EH PPO 1 upd (修复后) | 0.6480 | 0.1783 | 0.0255 |

**修复后验证结果（第二轮 strict paired, streaming GRU eval, zero-init ckpt, 2026-07-14 晚）**：
| | steady_P_D | weak3 | worst | PPO ratio |
|---|---|---|---|---|
| DAgger baseline (streaming GRU) | 0.6929 | 0.2825 | 0.0290 | — |
| Full PPO 1 upd | 0.6850 | 0.2621 | 0.0309 | OK (7e-6) |
| EH PPO 1 upd | 0.6918 | 0.2794 | 0.0267 | OK (5e-6) |

**第二轮改进**：(1) checkpoint 兼容 (zero_init 消除 pd_hist_proj 随机污染)；(2) streaming GRU 评估（与 rollout recurrent 路径一致）；(3) 每个 case 独立 fresh trainer + 同一 snapshot 恢复；(4) 同一 rollout seed 和 test bank。

**第二轮结论**：streaming GRU 评估下 DAgger baseline 的 weak3 更高 (0.283 vs 0.180)，说明此前 h=0 评估低估了 DAgger 的真实性能。修复后 Full PPO weak3 降幅约 2%（vs 修复前 26%），EH 几乎不变（-0.3%）。GRU/PPO ratio 修复已确认有效，此前"PPO 系统性失败"的结论需要降级为："旧循环状态处理下 PPO 更新失效"。

**本报告其余内容的参考价值**：模块交换实验、PCA/CKA 分析、统计方法论仍然有效。但所有关于"PPO 破坏 DAgger"的定量结论（M0→M1 的 Δweak3 等）都是在 ratio 非法条件下测得的，不可直接用于指导后续设计。

**建议**：用修复后代码重新跑一轮完整的模块交换 + 消融实验，在新的正确 baseline 上重新评估 EH/Full PPO 的长期表现。

## 一、审计动机

单次 PPO 更新从 DAgger 初始化后，steady 仅下降 ~3%，但 weak3 下降 ~26%。需要确定**哪个模块首先偏离了 DAgger 行为**，以及该效应是否具有统计显著性。

---

## 二、审计方法详解

### 2.1 配对模型

| 模型 | 定义 | 参数哈希 |
|------|------|------|
| M0 | DAgger frozen (5-iteration BC) | `8991796374291485308` |
| M1 | M0 + 1 PPO update (LR=3e-4, clip=0.1) | 记录于日志 |

### 2.2 方法 1：模块交换（Module Swap）

**原理**：将模型拆为三个独立可交换模块，构造 2³=8 个混合模型。

| 模块 | 包含参数 | 参数量 |
|------|------|:---:|
| **Encoder** | `self_enc`, `target_enc`, `neighbor_gru`, `neighbor_proj`, `global_enc` | ~47K |
| **Attention** | `attn.in_proj_*`, `attn.out_proj.*`, `attn_norm.*` | ~6 tensors |
| **Heads** | `dp_head`, `comm_head`, `role_head`, `dp_log_std`, `comm_proj`, `gate`, `intent_head` | ~10K |

**操作**：将 M0/M1 的模块参数按指定组合复制到新模型，在同 test bank 上评估。

**局限性**：拼接不同 checkpoint 的模块可能**人为制造接口不匹配**——不能区分"模块坏了"和"模块之间不兼容"。

### 2.3 方法 2：单模块更新消融（Per-Module Update Ablation）

**原理**：从同一个 M0 checkpoint 出发，使用相同 rollout、相同 advantage，只允许特定模块执行一次 PPO 更新。这是**最强因果证据**。

**操作步骤**：
1. 从 M0 复制完整参数快照 (sd_dagger)
2. 设置目标模块的 `requires_grad=True`，其余 `False`
3. 从相同环境 seed (123) 生成 rollout
4. 执行一次完整 PPO update (collect_rollout + update)
5. 在同一 test bank 上评估

**优势**：区分"哪个模块更新后会退化"（消融）vs"哪个模块换了会退化"（交换）。

### 2.4 方法 3：PCA + Effective Rank + CKA

**原理**：比较 M0 和 M1 在相同输入下的中间激活。

**指标**：
- **Effective Rank**：`exp(-Σ p_i log p_i)`，衡量表示多样性
- **90%/95% 方差维度**：累积方差达到阈值所需的主成分数
- **CKA**：Centered Kernel Alignment，衡量两个表示空间的整体相似性

**样本**：30 episodes × 50 frames × 8 UAV = 12,000 target embedding vectors。

### 2.5 统计方法：Paired Bootstrap

**原理**：对每个 test episode 计算配对差值 Δ_e = weak3_e^treated - weak3_e^baseline，对 {Δ_e} 做 5000 次有放回重采样，取 2.5%/97.5% 分位数作为 95% CI。

**判定**：CI 不包含 0 → 显著；CI 包含 0 → 不显著。

## 四、统计功效分析

| Test bank 规模 | per-ep weak3 std | 最小可检测效应 (80% power) | 可检测? |
|:---:|:---:|:---:|:---:|
| 20 episodes | 0.252 | ~0.16 | 仅 >0.16 的效应 |
| 100 episodes | 0.252 | ~0.071 | 仅 >0.07 的效应 |
| 400 episodes | 0.252 | ~0.035 | 可检测 ~0.03 效应 |

**结论**：per-ep weak3 方差 (0.252) 是主要限制因素。即使 100ep，单模块效应 (~0.01-0.03) 仍在检测阈值以下。EH 的 +0.028 恰好超过 100ep 的检测限。

## 五、已确认 vs 已排除

| 假设 | 证据 | 验证级别 | 判定 |
|------|------|:---:|:---:|
| Encoder 表示塌缩 | CKA=0.998, EffRank Δ=-0.3% | 🟡 | **排除** |
| 单模块是独立故障源 | 20ep: CI 跨零 + 符号翻转; 100ep: 效应 <0.07 | 🟢 | **排除** |
| A-only 是第一轮最大负向 | 第二轮符号翻转为 +0.006 | 🟢 | **排除** |

| PPO fine-tuning 系统性破坏 | 4 路径 (全量/KL/残差/CVaR) 全部退化 | 🟢 | **确认** |
| 模块交换 E×A 耦合 | 2³ 矩阵: E1A1H0 恢复至 0.738 | 🟡 | **待复现** |

## 六、最终结论

**🟢 确认**：单次 PPO 更新的模块级效应 < 评估噪声 (0.07 @ 100ep)。退化是跨模块的分布式、累积性漂移。单模块归因路线（"Attention 是根因"）在第一轮误导后在第二/三轮被推翻。

**🟢 确认**：EH (冻结 Attention + 更新 Encoder+Heads) 在 100ep paired bootstrap 下显著改善 weak3 (+0.028)。这是唯一通过多轮验证的显著配置。

**🟡 待验证**：EH 的长期稳定性 (仅测单次更新)。多步 EH fine-tuning 是否持续有效待 20+ep 训练验证。

**根本限制**：per-ep weak3 标准差 0.252 意味着即使 100ep，也只能检测 >0.07 的效应。更精细的模块级归因需要 ~400ep 或降低评估方差的方案。

---

## 七、算法方向

1. **DAgger frozen 作为最终基线** 🟢（多 bank 一致：0.71-0.74 steady）
2. **EH 安全微调** 🟢（100ep 显著 +0.028），长期稳定性 🟡 待验证
3. **停止全模块 PPO fine-tuning** 🟢（4 路径全失败，不可复现的单模块归因）
4. **如必须 fine-tune**：bounded residual (R2) 🟡（安全但保守）或 EH-only 🟢（显著但仅测单步）

## 附录 B：审计教训

1. **单次实验不可信** 🟢：A-only Δw3 从 -0.015 翻转为 +0.006。必须多轮 + paired bootstrap。
2. **paired bootstrap 必须** 🟢：无 CI 的均值比较在 20ep 下不可靠。
3. **模块交换 ≠ 因果** 🟢：交换的 "Encoder 77%" 被消融推翻——交换人为制造接口不匹配。
4. **效应量 < 噪声时停止归因** 🟢：单模块 Δw3 (~0.01) << per-ep std (0.25)。
5. **100ep 才获得显著结果** 🟢：EH +0.028 在 20ep 不显著 (p>0.05)，100ep 才显著。
6. **效应方向可翻转** 🟢：A-only 两轮符号相反，不可将单次方向作为结论。
